Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
и этот проект следует [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.0-beta] - 2026-07-20

### 🚀 Добавлено
- **Поддержка PMS5003** (датчик качества воздуха)
  - Чтение PM1.0, PM2.5, PM10
  - Новые MQTT-топики: `home/meteo/pm25`, `home/meteo/pm10`
  - Автоматическое обнаружение датчика при загрузке
- **Поддержка BME680** (экспериментально)
  - Измерение VOC, газа и качества воздуха
  - I2C адрес: 0x77 (конфликт с BME280, требуется переключение)
- **Веб-интерфейс с графиками** (экспериментально)
  - Отображение данных за последние 24 часа
  - Использование Chart.js для визуализации
- **OTA-обновления**
  - Возможность обновления прошивки по Wi-Fi
  - Веб-интерфейс для загрузки .bin файлов

### 🔧 Изменено
- Оптимизация использования RAM (на 15%)
  - Уменьшен размер буферов
  - Использование PROGMEM для статических строк
  - Оптимизация работы с JSON
- Улучшена обработка ошибок Wi-Fi
  - Автоматическое переподключение с экспоненциальной задержкой
  - Логирование ошибок в EEPROM
- Обновлены библиотеки до актуальных версий
  - Adafruit BME280: 2.2.2 → 2.3.0
  - PubSubClient: 2.8.0 → 2.9.0
  - ArduinoJson: 6.21.0 → 6.22.0

### 🐛 Исправлено
- Баг с переполнением буфера при отправке больших MQTT-сообщений
- Проблема с отображением на OLED после выхода из Deep Sleep
- Ошибка при сохранении настроек в EEPROM (некорректная запись)
- Исправлен memory leak в WiFiManager
- Исправлена проблема с чтением BME280 при низком напряжении

### ⚠️ Известные проблемы
- PMS5003 требует прогрева 30 секунд для точных показаний
- BME680 работает нестабильно на некоторых платах ESP32-C3
- Графики в веб-интерфейсе обновляются с задержкой 2-3 секунды
- OTA-обновления не поддерживают rollback

## [1.0.0] - 2026-07-15

### 🚀 Добавлено
- Первая стабильная версия
- **Поддержка BME280**
  - Температура (±1°C)
  - Влажность (±3%)
  - Давление (±1 гПа)
- **OLED-дисплей SSD1306**
  - Отображение текущих показаний
  - Циклический просмотр данных
  - Отображение IP-адреса и имени устройства
- **MQTT-клиент**
  - Подключение к любому MQTT-брокеру
  - Поддержка TLS (опционально)
  - Публикация в нескольких топиках
- **Веб-интерфейс**
  - Отображение данных в реальном времени
  - Настройка Wi-Fi и MQTT
  - Кнопка сброса настроек
- **EEPROM**
  - Сохранение настроек (Wi-Fi, MQTT, имя устройства)
  - Автоматическая загрузка при старте
- **Режим Deep Sleep**
  - Потребление < 20 мкА
  - Настраиваемый интервал пробуждения
  - Сохранение данных в RTC память
- **Индикация статуса**
  - Светодиод для отображения состояния
  - Режимы: Wi-Fi подключен, MQTT подключен, ошибка

### 📝 Документация
- Полный README.md с инструкцией
- Схема подключения (Fritzing)
- Распиновка ESP32-C3
- Примеры кода для начинающих

### 🛠️ Технические детали
- Платформа: Arduino 2.0.14
- Плата: ESP32-C3 DevKitM-1
- Flash: 4MB
- RAM: 400KB (использовано 125KB)

2. Подключите ESP32-C3 через USB
3. Загрузите прошивку через esptool:
```bash
